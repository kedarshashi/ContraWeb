[edit] Description and Recent Changes

  • Some features that used to be optional (turbo mode; control enabling/disabling) are now required.
  • There are more different sizes of fonts that they must support.
  • CSS is not given to them; they must write it themselves.
  • Page shows a "heart" in top-left if they link to our provided JS files correctly.

 

[edit] Common IPL Issues

  • Q: Why do almost all of my frames drop down by 1 line?
    • A: You split on ===== instead of =====\n .
  • Q: When they choose an animation such as "exercise", how do I turn the string "exercise" into the contents of the variable named 'exercise'?
    • A: There's no way built into the JavaScript language to do that, but you may want to tell the student to look at animations.js. It has a global array ANIMATIONS that maps from string indexes to the text of the corresponding animation. For example, ANIMATIONS["exercise"] is the text for the exercising man animation. (The writeup briefly mentions this array.)
  • Q: Why does it reload my page when I click Start or Stop?
    • A: Don't use a form.

[edit] Grading Checklist

(bold means not covered by Style Checker):

  • external correctness
    • page appearance
      • we want to be strict on colors, fonts, form controls, and alignment
        • make sure form controls have the large bold font
          (the options in the select box are not bold in Chrome/Safari; this is okay)
        • make sure H1, textarea, form fieldsets are centered within the page; W3C images are at right
          (the legends on the fieldsets may be centered or left-aligned; either is okay)
      • be lenient about things like:
        • whether the H1 size is 28pt or 32pt
        • pixel-perfect lining up with the expected image
        • exact position of the W3C images not that important, as long as they are at the right side of the page
        • minor spelling / casing errors, such as font "medium" instead of "Medium"
    • animation / start/stop
      • Changing animations: Make sure you can choose an existing animation, CHANGE it slightly, then push Start and you'll see the changes. For example, load up their 'Exercise' from the select box, start/stop it, and then change it to the following, and re-run it to make sure it reflects the change: 
  o
 -#-
 _|_
=====
  o
  #
_/ \_
      • Animating arbitrary new text: Make sure you can type in arbitrary text into the text box and it'll still animate it. Such as: 
^_^
=====
~_^
=====
^_^
=====
^_~
    • Font size change while animation is running, make sure it works.
    • Turbo: Should take effect immediately; shouldn't start animation over or change frame showing in the textarea. Clicking the turbo button before and/or after the start button should not break anything.
    • Enabling/Disabling: initially and after animation, 'Stop' disabled; during animation, 'Start' and 'Animation' box disabled
    • Custom Animation: If they don't include it properly in BOTH formats (JS and text), -1.
  • internal correctness
    • avoids redundancy
      • MUST use the ANIMATIONS array (for everything but Custom); look for redundant code to set the text area animation to each animation
      • look for redundant code for changing the font size
        • -1: if (...) { font = 7pt; } else if (...) { font = 12pt; } ...
        • -1: function font7pt() {...} function font12pt() {...} function font24pt() {...}
        • OK (no deduction): putting font size rules in CSS file, e.g.: .tiny {font-size: 7pt;} ... as long as JS code is short (one-liner) to set the corresponding CSS class on the textarea (no 7-way if/else in JS)
      • look for redundant timer start/stop code in several places
      • possibly look for redundant enabling / disabling code in several places (only if it's really ugly)
    • animation, timers, DOM used appropriately
      • -1 if they use setTimeout over and over instead of setInterval
      • obtrusive JS (onclick="...") is okay; but no multiple JS statements in one HTML onXXX handler string; no declaring variables in onXXX handler string; must be short; etc. (onclick="setAnimation(exercise);" is okay)
    • comments: They should have a good header at top, a heading on each function, AND some comments on complex code.
